# Cloudflare D1 Database

Managed SQLite for Workers. Use it for relational queries and stronger consistency than KV, and model many small databases rather than one shared cluster.

## Best Fit

- Per-user, per-tenant, or per-entity data sets.
- Relational queries, secondary indexes, and SQL migrations.
- Recovery needs covered by 30-day Time Travel.

## Core Properties

| Property | Detail |
|----------|--------|
| Query model | SQLite-compatible SQL |
| Scale unit | 10 GB maximum per database |
| Recovery | 30-day Time Travel |
| Access | Worker binding and HTTP API |
| Limits | 1 MB row, 30s query timeout, 10,000-statement batch |
| Pricing | Query and storage based |

## Quick Start

```bash
wrangler d1 create <database-name>
wrangler d1 execute <db-name> --remote --file=./migrations/0001_schema.sql
wrangler dev
```

## Query API

```typescript
// .all() returns all rows; .first() returns the first row or null
// .first(column) returns one value; .run() writes; .raw() returns array rows
const { results, success, meta } = await env.DB.prepare('SELECT * FROM users WHERE active = ?').bind(true).all();
const user = await env.DB.prepare('SELECT * FROM users WHERE id = ?').bind(userId).first();
```

Use `batch()` for multiple independent statements in one round trip. It is the main transaction primitive exposed by the Worker binding.

## Batch Operations

```typescript
const results = await env.DB.batch([
  env.DB.prepare('SELECT * FROM users WHERE id = ?').bind(1),
  env.DB.prepare('SELECT * FROM posts WHERE author_id = ?').bind(1),
  env.DB.prepare('UPDATE users SET last_access = ? WHERE id = ?').bind(Date.now(), 1),
]);
```

## Essential Commands

```bash
wrangler d1 create <db-name>
wrangler d1 list
wrangler d1 delete <db-name>
wrangler d1 execute <db-name> --remote --command="SELECT * FROM users"
wrangler d1 execute <db-name> --local --file=./migrations/0001_schema.sql
wrangler d1 export <db-name> --remote --output=./backup.sql
wrangler d1 time-travel restore <db-name> --timestamp="2024-01-15T14:30:00Z"
wrangler dev --persist-to=./.wrangler/state
```
